## 4. Public Critique Intake Canonical

All Wave 1 critique should be submitted through public GitHub Issues using the templates below:

```text
.github/ISSUE_TEMPLATE/
├── constitutional-critique.md
├── replay-receipt-critique.md
├── communications-protocol-critique.md
├── failure-mode-critique.md
└── accessibility-offline-redress.md
```

Each template includes:

- critique focus
- reproduction steps, if applicable
- constitutional boundary checks
- replayability notes
- optional attachments

This ensures:

- no hidden state
- no private authority channel
- no off-repo canonical feedback
- all critique is publicly contextualized

## 5. Private Notes Optional

Reviewers may send private notes only if necessary, such as sensitive examples.

Private notes must be summarized publicly into the Wave 1 feedback ledger:

```text
/wave-1/feedback/YYYY-MM-DD_summary.md
```

This preserves transparency, replayability, and public accountability.

Wave 1 reviewers are asked to:
