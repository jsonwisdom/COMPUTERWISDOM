Trigger Deck Standard

  • Each card uses unique TD### ID
  • Two-layer visual protocol: _front.png and _back.png
  • eJSON file required per card with all metadata
  • Snapback renderer loads and flips on click

Deployment Guidelines

  • Upload new cards to snapback/images/
  • Update deck.json with metadata and ID list
  • Back-layer files should include logo and standard footer text
