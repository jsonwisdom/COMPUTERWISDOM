# OPACITY_INDEX_V0

**Replayable Manic – Opacity Index**  
**Version:** 0.0  
**Status:** Coherently Invalid (Public Observability Metric)

## Purpose

The Opacity Index measures **replayability gaps** in public institutional and governance processes. It is **not** a morality, effectiveness, or political scoring system.

## Axes & Weighting

| Axis | Question | Weight |
|------|----------|--------|
| Chronology Visibility | Can event/decision evolution be replayed? | 20% |
| Source Accessibility | Are primary documents publicly reachable and stable? | 18% |
| Redaction Density | How much key information is obscured? | 15% |
| Attribution Clarity | Are decision-makers and processes identifiable? | 15% |
| Correction Visibility | Are revisions and updates clearly preserved? | 12% |
| Oversight Surface | Are lawful challenge pathways visible? | 10% |
| Machine Readability | Is the data structurally inspectable? | 5% |
| Confidence Disclosure | Are uncertainty bounds explicitly exposed? | 5% |

## Methodology Requirements

- All inputs must be T1/T2 public sources.
- Every release must include:
  - Per-axis breakdown
  - Composite confidence interval
  - Full methodology version and changelog
  - Link to complete Replay Thread
- The Index itself is versioned, forkable, and replayable.
- “Every map is incomplete” disclaimer required on all publications.

## Update Process

Updates occur via Replay Threads under the MANIC_RUNTIME ruleset. Competing forks are encouraged and must remain visible.
