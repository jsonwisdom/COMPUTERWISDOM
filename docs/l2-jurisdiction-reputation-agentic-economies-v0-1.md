## 2. L2 Jurisdiction

L2 Jurisdiction means the bounded domain where a receipt, claim, dispute, escalation, or agent action is allowed to operate.

Jurisdiction is a routing constraint, not a truth claim.

```json
{
  "l2_jurisdiction": true,
  "scope_bound": true,
  "truth_claim": false,
  "authority": false
}
```

### Allowed Jurisdiction Functions

- identify scope
- route to the proper registry
- distinguish local, institutional, chain, and civic surfaces
- prevent out-of-scope authority drift
- preserve replay lineage

### Must Not Claim

L2 Jurisdiction must not claim:

- universal authority
- legal finality
- truth by venue
- correctness by chain
- legitimacy by dashboard visibility

## 3. L2 Reputation

L2 Reputation is a replay-visible record of behavior, receipts, handoffs, commitments, disputes, and historical consistency.

Reputation is not morality.
Reputation is not guilt.
Reputation is not universal trust.

It is a structured surface for observed behavior over time.

```json
{
  "l2_reputation": true,
  "observable_history": true,
  "moral_judgment": false,
  "authority": false
}
```

### Reputation Inputs

- completed receipts
- disputed receipts
- escalation records
- continuity entries
- amendment lineage
- drill results
- agent commitments
- fulfillment evidence

### Reputation Outputs

- replay summaries
- consistency indicators
- unresolved-gap visibility
- custody trail visibility
- historical behavior context

### Must Not Claim

L2 Reputation must not claim:

- moral worth
- permanent trustworthiness
- criminality
- compliance
- institutional approval
- identity finality

## 4. L2 Agentic Economies

L2 Agentic Economies are machine-assisted economic workflows where agents may observe, route, propose, bid, escrow, reconcile, or report within strict authority limits.

Agents may operate at machine speed, but they may not create final authority.

```json
{
  "l2_agentic_economies": true,
  "agent_observation": true,
  "agent_proposal": true,
  "agent_execution_without_authority": true,
  "final_authority": false
}
```

### Allowed Agent Functions

Agents may:

- observe opportunities
- route receipts
- draft proposals
- compare obligations
- flag gaps
- prepare escrow metadata
- reconcile state
- emit replay artifacts
- request human or institutional review

Agents may not:

- assert truth beyond evidence
- silently mutate commitments
- decide disputes
- override human/institutional authority
- convert reputation into punishment
- convert jurisdiction into control
- convert economic routing into legal authority
