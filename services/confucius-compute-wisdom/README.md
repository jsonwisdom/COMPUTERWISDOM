## Placement Decision

The original mockup remains in `candidates/confucius_compute_wisdom_widget_candidate.html` because it is not live.

This folder exists so the idea can be reused throughout the repo without pretending the candidate widget is already production infrastructure.

Use this service surface for:

- builder code review flows
- claim tagging flows
- artifact repair reports
- receipt generation stubs
- Merkle purpose keys
- DeepSeek-assisted skill sessions
- public website modules after promotion

## Service Loop

1. Builder submits code, spec, prompt, receipt, or workflow.
2. DeepSeek or another model assists with analysis.
3. Compute Wisdom converts the analysis into structured claim tags and repair steps.
4. Jay's Wisdom decides what may be promoted.
5. A receipt or Merkle key records the purpose and evidence.

## Output Contract

A Wisdom Report should identify:

- input type
- apparent purpose
- claims detected
- missing receipts
- risk notes
- repair path
- suggested tests
- candidate receipt status

## Guardrails

- Do not mutate `portal.html` from this folder.
- Do not imply live backend execution.
- Do not grant badges from analysis alone.
- Do not call a candidate widget a production service.
- Do not ask builders to paste secrets, private keys, tokens, credentials, or sensitive personal data.
